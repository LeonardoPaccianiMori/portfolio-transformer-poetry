# Sonnet Corpus Scaling Generation Comparison

This report compares the selected V4 and V5 corpus-scaling fine-tuning runs.
It separates directly comparable shared-test evidence from fixed-prompt output
diagnostics and an assistant-generated qualitative assessment.

## Controlled Generation Protocol

| Setting | V4 | V5 |
| --- | --- | --- |
| Parent checkpoint | SwiGLU parent, step 110,000 | SwiGLU parent, step 110,000 |
| Selected fine-tuning checkpoint | step 1,250 | step 2,250 |
| Saved BPE vocabulary | 8,004 tokens | 8,004 tokens |
| Prompts | 5 fixed prompts | the same 5 fixed prompts |
| Base seed | 1,337 | 1,337 |
| Temperature / top-k | 1.0 / none | 1.0 / none |
| Maximum new tokens | 900 | 900 |
| Line control | 14 non-empty lines | 14 non-empty lines |
| Stop handling | suppress `<|endoftext|>` until 14 lines | the same |

Both batches preserve every prompt and reach the decoder-enforced 14-line
target. The line count therefore validates identical decoding control, not
learned sonnet structure.

## Shared Held-Out Test Result

The direct V4/V5 comparison uses the 99 poems held out in both corpus versions
and the same 23,738 BPE next-token targets.

| Arm | Shared test loss | Shared test perplexity |
| --- | ---: | ---: |
| V4 | 3.0643 | 21.419 |
| V5 | 3.0351 | 20.803 |

V5 has lower shared-test loss by 0.0292 nats per token and lower loss on 58 of
the 99 shared poems. This is a modest paired-test improvement, not a general
statistical claim. Full evidence: `reports/sonnet_corpus_scaling_shared_test.md`.

## Fixed-Prompt Diagnostics

| Measurement | V4 | V5 |
| --- | ---: | ---: |
| Average generated characters | 568.6 | 517.0 |
| Average non-empty lines | 14.0 | 14.0 |
| Average repeated character 4-gram ratio | 0.0930 | 0.0684 |
| Average unique-character ratio | 0.0730 | 0.0758 |
| `<|endoftext|>` occurrences | 0 / 5 samples | 0 / 5 samples |
| Low memorization-risk labels | 5 / 5 | 5 / 5 |
| 40-character training n-gram containment | 0.0000 for all samples | 0.0000 for all samples |

V5 is less repetitive by the character-4-gram diagnostic and produces slightly
shorter completions under the same 14-line target. The nearest-training-text
checks only show that neither batch has a strong surface-copying signal against
its own training corpus; they do not prove absence of memorization.

## Qualitative Assessment

The completed reviews rate all ten samples low for learned sonnet structure,
language/style plausibility, and coherence. V4 has medium repetition concerns
for two prompts; V5 is rated low for repetition on all five, consistent with the
automatic diagnostic. Neither arm produces sustained grammatical Italian,
coherent poetic thought, a reliable rhyme plan, or metre. V5 contains some
smoother local fragments, but the improvement does not make the outputs
realistic sonnets.

These assessments were generated by the project assistant from direct text
inspection and are not independent human judgments. The full samples and
per-prompt notes are retained in
`reports/qualitative_review_sonnet_corpus_scaling_v4_best.md` and
`reports/qualitative_review_sonnet_corpus_scaling_v5_best.md`.

## Conclusion

Increasing the sonnet corpus from V4 to V5 improves matched held-out likelihood
slightly and reduces visible short-pattern repetition under this decoding setup.
It does not solve the primary quality failure: the 33.7-million-parameter model
still lacks enough data, representation capacity, or training signal to sustain
grammatical and semantically coherent classical Italian sonnets. The next model
quality work should therefore target the broader data and training strategy,
rather than claiming that the V5 fine-tuning corpus alone is sufficient.
